Recap

Dependency Injection is a way to provide dependencies to your code instead of hard-coding them.

Comparison

Without Dependency Injection

  • The Person class creates everything it needs inside its constructor.
  • The Person class is brittle, inflexible, and hard to test.
  • This Person needs a profession. Instead of asking for it, the Person constructor instantiates its own copy from the very specific class Profession.
  • If the Profession class evolve and requires a parameter, we will break the Person class.
  • You have no control over the person's hidden dependencies.

With Dependency Injection

  • The Person class no longer creates a profession. It just consumes it.
  • If someone extends the Profession class, it is not a Person's problem.
  • You have complete control of Person's dependencies.
